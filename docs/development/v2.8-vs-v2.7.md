# v2.8 vs v2.7 Upgrade Notes

<!-- markdownlint-disable MD060 -->

Scope: Analysis-quality runtime behavior, CI gates, calibration, and UI/API status handling.
Date: 2026-03-17 (America/New_York)

---

## Summary

v2.8 adds runtime quality controls and release-governance checks that were not fully enforced in v2.7.
The upgrade focuses on active-classification safety, deterministic failure persistence, benchmark regression controls, and calibration automation.

---

## Comparison Matrix

| Area | v2.7 behavior | v2.8 behavior |
|------|---------------|---------------|
| Active classification | Validation strictness primarily centered on schema-level structure | Multi-stage gate path (data completeness, semantic validation, critique score gate) before active classification |
| Failed run persistence | Failures could be represented inconsistently across paths | Deterministic non-active states: `inactive_quality_failed`, `inactive_data_unavailable`, `inactive_schema_failed` |
| Failure metadata | Partial/varied failure detail by code path | Standardized failure metadata (`failure_code`, `failed_checks`, `missing_fields`, `retry_eligible`, `retry_after`, `validator_version`) |
| Semantic logic checks | Limited hard-stop semantic checks | Explicit semantic validator module with hard-fail policy for logic-inconsistent outputs |
| Critique quality control | No enforced section-score gate path | Enforced (env-gated) critique section-score thresholds with block/non-active/disabled-path test coverage |
| Benchmark regression guard | No CI merge policy tied to benchmark thresholds | CI benchmark gate with FAIL/WARNING/PASS threshold policy and artifact publishing |
| Calibration loop | No integrated scheduled update workflow | `run_calibration_update.py` plus scheduler task integration and replay test coverage |
| Template version routing | Static/default template routing | Runtime version pinning with global and per-skill env controls |
| UI status handling | Generic handling for declined/error/completed states | Inactive-state filtering and display support for non-active analysis statuses |

---

## Runtime and Policy Additions

1. Non-active status taxonomy and deterministic persistence path.
2. Semantic hard-fail validation path before active classification.
3. Critique section-score gate with thresholded section checks.
4. Guarded persistence behavior that blocks active classification but preserves source-of-truth artifacts.

---

## CI and Observability Additions

1. Benchmark gate module and policy thresholds.
2. KPI aggregation and benchmark report generation scripts.
3. CI workflow artifact publishing for quality evidence.
4. Dashboard runbook guidance for alert thresholds and remediation.

---

## Scheduler and Calibration Additions

1. Calibration report and priors generation script.
2. Scheduled `run_calibration_update` task support in orchestrator dispatch.
3. Replay-based test coverage for calibration update behavior.
